01 Schedule Trigger TRIGGER
Automatically wakes the workflow every 6 hours — at midnight, 6am, noon, and 6pm. No human interaction needed.
Node type: n8n Schedule Trigger
Interval: Every 6 hours
Output: Timestamp + date/time metadata
→ Passes timestamp to the next node
↓
🤖
02 AI Agent — Search All Companies AI AGENT
The brain of the system. Uses Claude Sonnet 4.6 + SerpAPI to actively search the internet for job postings at all 7 companies across all portals. Returns structured JSON.
Node type: n8n AI Agent
Model: Claude Sonnet 4.6 (via Anthropic Chat Model sub-node)
Tool: Google Search via SerpAPI sub-node
Prompt source: Define below (not chat trigger)

What Claude searches:
→ LinkedIn Jobs, Internshala, Wellfound, Naukri
→ Indeed India, Glassdoor, official career pages
→ Company LinkedIn posts & announcements

Output format: Raw JSON array of job objects
Each object has: job_id, company, title, type, department,
location, mode, duration, stipend, deadline, apply_link,
source, eligibility{degree, branches, skills, tools...}
↓
⚙️
03 Parse Job Results CODE
Converts Claude's raw text output into clean structured data. Strips markdown formatting, parses JSON, and splits the array so each job flows through the pipeline individually.
Node type: n8n Code (JavaScript)

Logic:
1. Extract text from Claude's response
2. Strip ```json fences if present
3. Parse as JSON array
4. If empty → return {no_new_jobs: true}
5. If has items → split into individual items

Why needed: Claude returns text; n8n needs
structured JSON objects to route between nodes
↓
🔀
04 Any Jobs Found? IF / BRANCH
First decision gate. If Claude found zero openings this run, the workflow ends silently — no email, no noise. Only real results continue.
Node type: n8n IF node
Condition: Is no_new_jobs === true?

TRUE branch → "No New Jobs — End Silently"
FALSE branch → Continue to deduplication

Why silent end? You don't want an email
every 6 hours saying "nothing found"
Decision point ● TRUE path ● FALSE path
✓ NO JOBS FOUND
Goes to "End Silently" node → workflow stops quietly. No email sent. Runs again in 6 hours.
→ JOBS FOUND
Continues to Google Sheet deduplication check for each job found.
↓ (jobs found path continues)
📊
05 Read Seen Job IDs GOOGLE SHEETS
Reads your entire SeenJobs Google Sheet and loads all previously alerted job IDs into memory. This is how the system knows what it's already told you about.
Node type: Google Sheets → Get Row(s)
Document: SeenJobs (from list)
Sheet tab: Sheet1
Credential: Google Sheets account 3

Output: Array of all rows, each with job_id,
company, title, date_found etc.

The SeenJobs sheet columns:
job_id | company | title | type | location
apply_link | match_score | recommended_action
deadline | date_found
↓
🔍
06 Check If New CODE
Compares this job's unique ID against all IDs in the Google Sheet. If it's been seen before → mark as not new. If it's genuinely new → flag it to continue.
Node type: n8n Code (JavaScript)

Logic:
1. Build a Set of all seen job_ids from sheet
2. Check if current job's job_id is in Set
3. Return {is_new: false} if already seen
4. Return {is_new: true, job: {...}} if new

Job ID format:
company-role-YYYYMM
e.g. "ideaforge-uav-intern-202506"

Why a Set? O(1) lookup — fast even with
hundreds of previously seen jobs
↓
🔀
07 Is New Opportunity? IF / BRANCH
Second decision gate. Already alerted jobs are dropped here. Only genuinely new listings proceed to AI scoring and email.
Node type: n8n IF node
Condition: is_new === true

FALSE → Stop (already sent email for this job)
TRUE → Continue to profile match scoring

This is the deduplication gate. Without
this, you'd get the same email for the
same job every 6 hours forever.
↓ (new job path)
🧠
08 Claude — Score Profile Match AI AGENT
A second Claude call that reads the job details and scores it against your specific engineering profile — ME background, UAV experience, Arduino, CAD, embedded systems, GitHub portfolio.
Node type: n8n AI Agent
Model: Claude Sonnet 4.6

Your profile Claude scores against:
→ Mechanical Engineering undergraduate
→ UAV, drones, aerospace, motorsports interest
→ Arduino-based flight controller experience
→ CAD assembly and engineering design
→ Embedded systems exposure
→ Active GitHub project portfolio

Output JSON:
match_score: 1–10
why_suitable: "2-3 sentence explanation"
missing_skills: ["ROS", "Python"]
suggestions: "Add ROS project to GitHub"
recommended_action: "Apply immediately"
↓
🔗
09 Merge Job + Score CODE
Combines the job data from the search step with the scoring data from Claude into one unified object, ready to be formatted into an email.
Node type: n8n Code (JavaScript)

Logic:
1. Get job object from step 7
2. Parse Claude's scoring output
3. Merge: {...job, ...score}
4. Pass unified object forward

Why separate? The search and scoring
are two different AI calls. This merges
them back into one clean data object.
↓ (splits into two parallel paths)
📧

Appends the job to SeenJobs sheet so it's never emailed again. Runs in parallel with the email.

Document: SeenJobs
Mode: Map Automatically

Columns written:

location, apply_link, match_score
recommended_action, deadline
date_found (timestamp)
